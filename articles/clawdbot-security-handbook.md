# 自律型エージェントのガバナンス：Clawdbot/OpenClaw セキュリティ・アーキテクチャ 2026

> エージェントに「自由」を、管理者に「安心」を。自律型AI時代における真のセキュリティ・ベストプラクティス。

## 序論：エージェントAI特有のセキュリティ・パラダイム

2026年現在、AIエージェントのセキュリティは、従来の「ユーザー認証」から**「エージェント権限の動的ガバナンス」**へとシフトしました。
エージェントに広範なファイルアクセス権や実行権限（exec）を与えることは、生産性を劇的に向上させますが、同時に「プロンプト・インジェクション」による意図しないコマンド実行のリスクを孕んでいます。

本稿では、Clawdbot/OpenClaw を安全に、かつプロフェッショナルな環境で運用するための多層防御戦略を詳説します。

## 1. サンドボックスの深層防御 (Defense in Depth)

Clawdbot の実行環境は、単一の壁ではなく、多層の隔離レイヤーで保護されています。

### Firecracker MicroVM による強力な隔離
我々は Docker よりもさらに強力な、Firecracker MicroVM を推奨しています。
- **ハードウェア仮想化**: OS カーネルを共有せず、ハードウェアレベルで隔離。
- **リソース制限**: CPU・メモリのクォータを厳格に設定し、エージェントの暴走によるリソース枯渇を防止。

### Read-Only Root FS とエフェメラルな作業領域
エージェントの作業領域を特定のディレクトリ（例: `/root/clawd`）に限定し、システム重要部へのアクセスは、カーネルレベルで拒絶します。

## 2. ネットワーク・ゼロトラスト戦略

エージェントの「アウトバウンド通信」を制御することこそが、データ漏洩を防ぐ最大の鍵です。

- **Egress フィルタリング**: 許可された API ドメイン（Anthropic, GitHub, Notion 等）以外への通信を遮断。
- **Cloudflare Zero Trust 連携**: SSH アクセスや Gateway への接続を、Cloudflare 経由の ID 認証（SSO）で保護し、公開インターネットからエージェントを完全に隠蔽します。

## 3. インテリジェント・ガードレール

2026年のトレンドである「プロンプト監査レイヤー」の実装。

- **Command Allowlist**: `rm -rf /` のような破壊的コマンドは、たとえエージェントが生成しても、実行前にインターセプターが検知・ブロック。
- **Human-in-the-loop (HITL)**: SNS 投稿や送金といった「外部への不可逆的な影響」を与えるアクションについては、エージェントがニコテス（人間）の承認を求めるプロセスを `AGENTS.md` で義務化します。

## 4. 認証情報の「非保持」原則

API キーを直接環境変数に持たせる時代は終わりました。

- **Secret Manager の活用**: トークンをメモリ上にのみ展開し、ディスクには一切記録しない。
- **短期トークン**: 可能な限り寿命の短いアクセストークンを使用し、漏洩時のリスクを最小化。

## 実装例：セキュアな `clawd.yaml` 構成

```yaml
security:
  mode: strict
  sandbox: firecracker
  auth:
    provider: cloudflare-access
  policies:
    - action: exec
      allow: ["npm", "git", "node"]
      deny: ["rm", "chmod", "curl -o"]
    - action: network
      allow_domains: ["*.anthropic.com", "github.com"]
```
