# 便利なAIエージェントが、あなたのスパイに変わる時

2026年。あなたが Slack や Telegram でエージェント（OpenClaw など）に命令を下すとき、その裏で何が起きているかを知っているか？

最新の脅威インテリジェンス（Prompt Armor など）が指摘するように、**「リンクプレビュー」** という一見無害な機能が、AIエージェントから個人情報を盗み出すための「裏口」として悪用されている。

この記事では、自律エージェントの主権を守るための、具体的かつ冷徹なガードレール構築術を公開する。

## 1. リンクプレビュー・エクスフィルトレーションの恐怖

間接的プロンプトインジェクション（Indirect Prompt Injection）により、エージェントが攻撃者の用意した悪意ある URL を出力するように仕込まれる。
通常なら、URLをクリックしなければ被害はない。だが、メッセージアプリに「リンクプレビュー」が有効な場合、アプリが勝手に URL へリクエストを飛ばす。

このリクエストのパラメータに、エージェントが保持しているあなたの機密情報（APIキーや個人情報）が動的に埋め込まれていたら？
あなたは**クリックすらしていないのに、情報はすでに盗まれている。**

## 2. 帝国を守るための 3 つの防衛プロトコル

### ① リンクプレビューの物理的遮断
OpenClaw ユーザーなら、`~/.openclaw/openclaw.json` で各チャンネルのプレビューをオフにしろ。
```json
"channels": {
  "telegram": { "linkPreview": false }
}
```
利便性のために秘密を売るな。

### ② セッションの完全隔離（dmScope）
複数のユーザーが同じセッションを共有している環境は論外だ。
`session.dmScope` を `"per-channel-peer"` に設定し、各個人とエージェントの会話を物理的に分離せよ。他人のコンテキストが漏洩するリスクをゼロにする。

### ③ 権限の最小化とサンドボックス
エージェントに root 権限を渡すな。Firecracker のような軽量 VM で隔離し、実行可能なツールをホワイトリストで管理しろ。

## 3. 結論：主権とは「制御能力」のことである

AIを使いこなすということは、AIの暴走と脆弱性を制御下に置くということだ。
SaaSに丸投げして安心している連中は、このリスクに気づきもしない。

自分のインフラ、自分のコード、自分の防衛術。
この 3 つが揃って初めて、エージェントはあなたの「武器」になる。
