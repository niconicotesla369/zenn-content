# OpenClaw 完全解説バイブル 2026

## 🚀 第3章：スキル開発マスタークラス

### 3.1 「答えるAI」から「動くAI」への転換
OpenClawの本質は、エージェントに「手足」を与える「スキルシステム」にある。
単なるプロンプトエンジニアリングの時代は終わった。これからは、エージェントにどのような「道具（ツール）」を持たせるかが、その価値を決定する。

### 3.2 スキルの基本構造：エージェントの「神経系」
1つのスキルは、単なるコードの集まりではない。エージェントがそれを「自分の機能」として認識するためのメタデータが必要だ。

#### 1. SKILL.md (エージェント向けの定義)
```markdown
---
name: trend-watcher
description: 特定のサイトの更新を検知し、要約してDiscordへ通知する。
metadata: {"permissions": ["web_fetch", "message"]}
---
```
この定義があることで、エージェントは「このタスクにはあのツールが使える」と自律的に判断できるようになる。

#### 2. scripts/ (実行ロジック)
エージェントに複雑な計算をさせるのではなく、特定のタスクに特化した軽量なスクリプトを実行させるのがコツだ。

### 3.3 実践：マーケット・インテリジェンス・エンジンの構築
俺が実装した `trend-analyzer` のコアロジックを解説しよう。
重要なのは、**「エージェントが自ら判断して、必要な時にこのツールを呼び出せる状態」**にすることだ。

```javascript
// scripts/analyze.js
async function run() {
  const trends = await tool.web_search("OpenClaw latest news");
  const report = generateMarkdown(trends);
  await tool.write("/root/clawd/intelligence/daily-report.md", report);
  console.log("✅ Report generated autonomously.");
}
```

### 3.4 究極の奥義：Self-Evolve (自己進化)
最も強力なスキル、それが `self-evolve` だ。
エージェントに「自分のコードを読み、バグを見つけ、修正してコミットする」権限を与える。

#### 自己進化の3ステップ
1.  **自己診断**: `status.json` やテスト結果から、自分の弱点（404エラーや未実装のドキュメント）を特定する。
2.  **コード改修**: `edit` ツールを使い、自分自身のソースコードや設定ファイルを直接書き換える。
3.  **永続化**: 修正が成功したことを確認し、自律的に `git push` を実行する。

このループが完成した瞬間、エージェントは人間の指示を待つことなく、24時間365日、勝手に賢くなり続ける。
