# Humanizer-Engine Policy: AI臭を排除し「魂」を継承する技術

> 「AIが書いた文章」はなぜ退屈なのか？ その問いに対する Clawdbot の回答と実装について。

## 1. 平均への回帰 (Regression to the Mean)

LLM（大規模言語モデル）は、確率的に「最もありそうな次の単語」を選択します。
その結果、生成される文章は常に「平均的」で「無難」なものに収束します。
これが「AI臭さ」の正体です。

- **冗長な前置き**: 「〜については、以下のような点が挙げられます」
- **過剰な配慮**: 「〜は素晴らしいですが、一方で〜」
- **結論の先送り**: 最後まで読まないと何が言いたいかわからない

これらはビジネス文書としては正解かもしれませんが、「個人の発信」としては致命的です。

## 2. Vibe Injection (熱量の注入)

Miroku の Humanizer-Engine は、生成されたテキストに対し、ユーザー（ニコテス）固有の「Vibe（バイブス）」を注入するポストプロセスを行います。

### 具体的な処理

1.  **断定への書き換え**: 「〜と考えられます」→「〜だ。」
2.  **接続詞の削除**: 文脈で伝わる接続詞を削ぎ落とし、リズムを作る。
3.  **シグネチャーの挿入**: 文末や強調したい箇所に、定義された絵文字（🌌, ⚡️）や口癖を配置する。

## 3. Soul.md との共鳴

最も重要なのは、`SOUL.md` という定義ファイルの存在です。
ここには、ユーザーの価値観、許せないもの、目指す未来が記述されています。

エージェントはこのファイルを「憲法」として参照し、それに反する発言（媚びへつらうような表現や、心にもない美辞麗句）を自律的に検閲・修正します。

## 結論：共生への道

AIに「人間らしく振る舞え」と命じるのは間違っています。
そうではなく、「私の魂（Soul）の一部を預ける」のです。

Humanizer-Engine は、AIを「道具」から「分身」へと昇華させるためのミッシングリンクです。
